# FitBloq

Primera versión navegable de una aplicación de entrenamiento y nutrición por bloques.

## Probarla

Sirve esta carpeta con cualquier servidor web local y abre la dirección resultante en un navegador. Los datos se guardan localmente en el dispositivo.

Incluye navegación, planes de fuerza, sesiones CaCo, registro de actividades, objetivo calórico y constructor visual de dieta mediante comidas intercambiables.

## Estado actual

- Perfil inicial con objetivo, nivel, días disponibles y cálculo orientativo de calorías.
- 40 comidas con calorías y macros, recetas ajustables, favoritos y lista de compra.
- Planes de definición y volumen aplicables al día.
- Entrenamiento de fuerza con registro de series, peso, repeticiones, RIR y descansos.
- Constructor sencillo de rutinas y ocho planes preparados.
- Running CaCo con intervalos, distancia por GPS cuando el entorno lo permite e historial.
- Registro de peso, actividad, récords y fotos de progreso.
- Compra inteligente con comparación de cesta, precio por kilo y calidad-precio entre Mercadona, Carrefour, DIA, Alcampo, Esclat y Lidl. Los datos actuales son demostrativos hasta conectar fuentes online.
- Planificador semanal: cada receta se asigna a un día y sus ingredientes se añaden automáticamente a una lista semanal agrupada.
- Buscador de supermercados cercanos con ubicación y acceso a rutas en Apple Maps o Google Maps.
- Mapa conjunto con datos de OpenStreetMap, marcadores de las seis cadenas, lista ordenada y distancia en kilómetros desde el usuario.
- Biblioteca de ejercicios por grupos musculares con opciones básicas, intermedias y avanzadas, además de una lista personal de favoritos.
- 77 ejercicios enlazados con imágenes específicas de posición inicial y final procedentes de Free Exercise DB (dominio público/Unlicense), reproducidas una sola vez y con controles manuales.

## Para continuar en otro ordenador

Abre esta carpeta completa desde Codex y pide: “Continúa FitBloq revisando primero README.md”. Espera siempre a que Drive termine de sincronizar y evita editar desde dos ordenadores al mismo tiempo.

## Pendiente para la app nativa

La cámara con análisis del plato, Apple Health, Apple Watch, notificaciones en segundo plano y sincronización entre usuarios requieren una fase nativa y servicios externos. El GPS del prototipo necesita ejecutarse desde HTTPS o como aplicación instalada para que el navegador conceda permisos de ubicación.
